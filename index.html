Lessons

Lesson one

We covered what HTML is, which is a hyper markup language use to create the structure and render text unto a website. We also covered the basic structure of a web page and other elements and tags. below is list of some of the tags.

  • DOCTYPE html
  • html tag
  • meta tag
  • title
  • header tag
  • body

We also covered some block elements and inline element tags. Examples of block tags are heading tags, div, img etc. while examples of inline tags elements are span, strong, em etc.

We also installed some text editor, covered folder and file creation. installed some plugins to our editor to optimism workflow

Lesson two

This was a bit challenging as a bit more technical topics were covered. We basically downloaded git and created a github account. We covered repository creation, how initialize file tracking, pull and add commits. We also saw to working in teams is facilitated through main and branch repos. Repository cloning was also covered and som commit best practices too. The main commands to keep in mind are:

  • git init at the start of every project
  • git status to check status of files
  • git add
  • git commit
  • git checkout
  • git branch
  • git merge

Reflection so far

Starting this course has been both exciting and challenging. At first, I felt unsure about where to begin, even with basic HTML and CSS, but I quickly discovered how small changes could shape a webpage’s structure and style. Experimenting with images, colors, layouts, and simple effects taught me the basics of design and functionality, though I often ran into issues that required patience and trial-and-error.

Even in these early stages, I have started to see how coding can be creative, and how learning step by step builds confidence. I am still exploring and figuring things out, but I feel motivated to continue, knowing that each small experiment brings me closer to understanding how websites really work.

Learning Git and GitHub was confusing at first. The concepts of commits branches, and pushing changes felt overwhelming, and I often got stuck trying to connect my local files to GitHub. Gradually, with practice, I started understanding how version control works and how to track changes properly. It is still new to me, but I am beginning to feel more confident navigating repositories and managing my projects online.
